# Отчет по оптимизации проекта Accurate Shop (Cloudflare Pages)

## 1. Проблема
При попытке деплоя на **Cloudflare Pages** возникала ошибка превышения лимита размера бандла функций:
`Generated Pages Functions bundle size (43.4 MB) is over the limit of 25.0 MiB`.

Дополнительно билд падал из-за:
- Использования Node.js API (`fs`, `path`, `process.cwd`) в Edge Runtime.
- Ошибок типизации TypeScript в Next.js 16.
- Невалидных параметров в `next.config.ts`.
- Рассинхронизации `package.json` и `package-lock.json`.

## 2. Проведенные работы

### Оптимизация размера бандла (Bundle Size)
- **Консолидация API:** Объединили 6 отдельных API-роутов админки в один универсальный роут с поддержкой опциональных параметров: `src/app/api/admin/[[...slug]]/route.ts`. Это позволило инициализировать **Prisma Client** только один раз для всей админки, сэкономив около 10-15 МБ общего объема.
- **Отказ от Edge Runtime для страниц:** Убрали `runtime = "edge"` и `force-dynamic` из 18 файлов (главная, каталог, товары, админ-панель). Теперь эти страницы генерируются статически (SSG/ISR), что выводит их из-под лимита функций Cloudflare.
- **Удаление тяжелых SDK:** Полностью удалена библиотека `@aws-sdk/client-s3`. Загрузка файлов в R2 переведена на нативные биндинги Cloudflare (`process.env.BUCKET.put()`), что значительно облегчило зависимости.
- **Динамический импорт:** Компонент `MobileMenu` (использующий тяжелую `framer-motion`) переведен на `next/dynamic`, чтобы не раздувать начальный JS-бандл страниц.

### Исправление ошибок билда
- **TypeScript:** Исправлены типы для параметров `params` в асинхронных серверных компонентах и роутах (теперь используются `Promise` согласно требованиям Next.js 15/16).
- **Next.config:** Удалены невалидные свойства `appIsrStatus` и `optimizePackageImports`.
- **R2 Upload:** Переписан роут загрузки изображений — теперь он полностью совместим с Edge Runtime и не использует Node-специфичные модули.
- **Nodejs_compat:** В `wrangler.toml` добавлен флаг `compatibility_flags = ["nodejs_compat"]`.

## 3. Текущее состояние
- **Последний коммит:** `b29a776` ("fix: resolve Next.js build and type errors").
- **Зависимости:** `package-lock.json` синхронизирован, удалено более 3000 строк неиспользуемого кода зависимостей.
- **Локальная работа:** `pnpm dev` запускается корректно, ошибка с хостом `via.placeholder.com` для картинок исправлена.

## 4. Рекомендации для деплоя
Для успешного завершения деплоя необходимо:
1. Выполнить `git push origin main`.
2. В консоли Cloudflare Pages убедиться, что собирается именно последний коммит.
3. Если билд все еще будет превышать 25 МБ (что маловероятно), следующим шагом будет вынос Prisma в отдельный Worker через RPC или использование Cloudflare Hyperdrive.
